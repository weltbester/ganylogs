Tool to identify new syslog messages File\+: \mbox{\hyperlink{ganysyslogs_8c}{ganysyslogs.\+c}}

Author\+: Manfred Jaede \& Georg Pohl, 70174 Stuttgart

Create date\+: 8th of November 2020

Version\+: 1.\+0

Last Change\+: 13th of November 2020



 This program can be used to identify unknown syslog messages, when you introduce a new router software. It is meant to be used durch den Product Validation (PV) time only. Therefore you feed the program with the involved router hostnames, so that it can browse through the recent logfile and grep out every syslog message of the before defined hosts.

Every message is then compared with the \textquotesingle{}blacklist\textquotesingle{}-\/ and a \textquotesingle{}whitelist\textquotesingle{} file.


\begin{DoxyEnumerate}
\item If it is found in \textquotesingle{}blacklist\textquotesingle{}, the message is known but regarded as not-\/remarkable and the original message is skipped.
\item If it is found in the \textquotesingle{}whitelist\textquotesingle{} it is already know and declared as remarkable and therfore under control. So no further processing needed and therefore also skipped.
\item The message is neither known in the \textquotesingle{}blacklist\textquotesingle{} nor the \textquotesingle{}whitelist\textquotesingle{} means we are dealing with an unknown message here. This message is send to the initiator of the script via e-\/\+Mail for further investigation. Depending on the importance of the syslog message, it then copied either to the \textquotesingle{}black-\/\textquotesingle{} or \textquotesingle{}whitelist\textquotesingle{}.
\end{DoxyEnumerate}

\textquotesingle{}whitelist\textquotesingle{}\+: contains log messages which are importand and have to be transferred to the \char`\"{}\+M\+O\+N\+I\+T\+O\+R\+I\+N\+G S\+Y\+S\+T\+E\+M\char`\"{}.
